## Executive Summary

This document proposes a holistic PO (Purchase Order) lifecycle management system for the WMS, introducing a **PO Command Center** as the central hub for warehouse operations. The design prioritizes:

- **Visibility**: Real-time view of all POs across every stage
- **Human Gates**: Clear approval points with accountability
- **Efficiency**: Streamlined flow for 5-10 daily POs with same-day processing
- **Flexibility**: Configurable landing pages per role

## 1. Current State Analysis

### 1.1 Existing Pages
| Page | Purpose | Gap |
|------|---------|-----|
| `/orders` | List all orders | No lifecycle stage visibility |
| `/order-queue` | Processing queue | Limited to single workflow |
| `/reservations` | Manage reservations | Disconnected from fulfillment |
| `/manufacturing-orders` | Track MOs | No PO linkage visibility |

### 1.2 Identified Gaps

1. **No Holistic Dashboard**: Users must navigate multiple pages to understand PO status
2. **Hidden Human Gates**: Approval requirements scattered across pages
3. **No Timeline View**: Cannot see PO progression history
4. **No Stage-Based Queues**: Mixed statuses in single lists
5. **Limited Carrier Tracking**: No external carrier capture

## 4. Contract Amendments

### 4.1 Database Schema Changes

#### Orders Table Extensions

```sql
-- Lifecycle tracking columns
ALTER TABLE orders ADD COLUMN IF NOT EXISTS picking_started_at TIMESTAMPTZ;
ALTER TABLE orders ADD COLUMN IF NOT EXISTS picking_started_by UUID REFERENCES auth.users(id);
ALTER TABLE orders ADD COLUMN IF NOT EXISTS prepared_at TIMESTAMPTZ;
ALTER TABLE orders ADD COLUMN IF NOT EXISTS prepared_by UUID REFERENCES auth.users(id);
ALTER TABLE orders ADD COLUMN IF NOT EXISTS shipped_by UUID REFERENCES auth.users(id);

-- Carrier tracking (for shipped orders)
ALTER TABLE orders ADD COLUMN IF NOT EXISTS carrier_type TEXT CHECK (carrier_type IN ('internal', 'external'));
ALTER TABLE orders ADD COLUMN IF NOT EXISTS carrier_name TEXT;
ALTER TABLE orders ADD COLUMN IF NOT EXISTS tracking_id TEXT;

-- Invoice tracking
ALTER TABLE orders ADD COLUMN IF NOT EXISTS invoiced_by UUID REFERENCES auth.users(id);

-- Delivery confirmation
ALTER TABLE orders ADD COLUMN IF NOT EXISTS delivered_at TIMESTAMPTZ;
ALTER TABLE orders ADD COLUMN IF NOT EXISTS delivered_confirmed_by UUID REFERENCES auth.users(id);
```

#### Order Status Enum Extension

```sql
-- Extend order_status enum if needed
-- Current: PENDING, APPROVED, IN_PROGRESS, COMPLETED, CANCELLED
-- Proposed additions:
ALTER TYPE order_status ADD VALUE IF NOT EXISTS 'PICKING';
ALTER TYPE order_status ADD VALUE IF NOT EXISTS 'PREPARED';
ALTER TYPE order_status ADD VALUE IF NOT EXISTS 'SHIPPED';
ALTER TYPE order_status ADD VALUE IF NOT EXISTS 'DELIVERED';
ALTER TYPE order_status ADD VALUE IF NOT EXISTS 'INVOICED';
```

### 4.2 New Tables

#### Role Landing Page Settings

```sql
CREATE TABLE IF NOT EXISTS role_landing_settings (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  role TEXT NOT NULL UNIQUE,
  landing_page TEXT NOT NULL DEFAULT '/dashboard',
  allow_user_override BOOLEAN DEFAULT true,
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);

-- Default values
INSERT INTO role_landing_settings (role, landing_page) VALUES
  ('warehouse_staff', '/po-command-center'),
  ('warehouse_manager', '/po-command-center'),
  ('accounting', '/dashboard'),
  ('admin', '/dashboard')
ON CONFLICT (role) DO NOTHING;
```

#### User Landing Page Override

```sql
CREATE TABLE IF NOT EXISTS user_landing_preferences (
  user_id UUID PRIMARY KEY REFERENCES auth.users(id),
  landing_page TEXT NOT NULL,
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);
```
